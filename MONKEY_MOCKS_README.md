# Monkey Mocks with Mockery

This document explains how to use the **Monkey Mocks** pattern with Mockery to mock functions (not just interfaces) in Go unit tests.

## Overview

Mockery is designed to generate mocks for Go interfaces. However, when you need to mock standalone functions (like utility functions, package-level functions, or functions that aren't part of an interface), you can use the **Monkey Mocks** pattern.

The pattern works by:
1. Creating a dummy interface (`monkeyMethods`) that declares all the functions you want to mock
2. Using Mockery to generate mocks for this interface
3. Using the generated mock to replace function variables at runtime
4. Restoring the original functions after the test

## File Structure

```
your-package/
├── generate_mocks.go      # Contains monkeyMethods interface (only built with mocks tag)
├── monkey_test.go         # Contains Patch() and Unpatch() methods
├── monkey_mocks_test.go   # Auto-generated by mockery (don't edit)
└── your_test_file.go      # Your actual test file
```

## Step-by-Step Implementation

### 1. Create `generate_mocks.go`

Create a file with the `//go:build mocks` build tag that defines the `monkeyMethods` interface:

```go
//go:build mocks

package yourpackage

import (
    "context"
    // ... other imports
)

type monkeyMethods interface {
    // List all functions you want to mock
    InternalUtilGetCallbackToken() (string, error)
    InternalUtilGetSignedToken(projectNumber string) (string, error)
    InternalUtilGetPairedRegionURI(region string) (string, error)
    validateReplicationResourceId(ctx context.Context, projectNumber string, paramReplicationResourceId string, paramsVolumeResourceId string, se database.Storage) error
    // ... add more functions as needed
}
```

**Important Notes:**
- Use `//go:build mocks` to ensure this file is only included when building mocks
- The interface name must be `monkeyMethods` (this is configured in `.monkeyMocks.yaml`)
- Include all function signatures you want to mock
- Function names and signatures must match exactly with the actual functions

### 2. Create `monkey_test.go`

Create a file that implements the `Patch()` and `Unpatch()` methods:

```go
package yourpackage

import (
    "github.com/vcp-vsa-control-Plane/vsa-control-plane/utils/auth"
    "github.com/vcp-vsa-control-Plane/vsa-control-plane/utils"
    // ... other imports for original functions
)

func (_m *monkeyMock) Patch() {
    // Replace function variables with mock implementations
    InternalUtilGetCallbackToken = _m.InternalUtilGetCallbackToken
    InternalUtilGetSignedToken = _m.InternalUtilGetSignedToken
    InternalUtilGetPairedRegionURI = _m.InternalUtilGetPairedRegionURI
    validateReplicationResourceId = _m.validateReplicationResourceId
    // ... add more function assignments
}

func (_m *monkeyMock) Unpatch() {
    // Restore original function implementations
    InternalUtilGetCallbackToken = auth.GetSignedAccessToken
    InternalUtilGetSignedToken = auth.GetSignedJwtToken
    InternalUtilGetPairedRegionURI = utils.GetPairedRegionURI
    validateReplicationResourceId = _validateReplicationResourceId
    // ... add more function restorations
}
```

**Important Notes:**
- `Patch()` assigns mock functions to function variables
- `Unpatch()` restores the original functions
- Function variable names must match exactly with the actual function variables in your package
- Import the packages containing the original functions

### 3. Configure Mockery

The `.monkeyMocks.yaml` file is already configured for the project:

```yaml
with-expecter: True
tags: mocks
packages:
  github.com/vcp-vsa-control-Plane/vsa-control-plane/core/orchestrator/replication:
    config:
      inpackage: True
      dir: "{{.InterfaceDir}}"
      filename: "monkey_mocks_test.go"
      mockname: "monkeyMock"
      include-regex: "monkeyMethods"
      recursive: True
```

**Key Configuration:**
- `include-regex: "monkeyMethods"` - Only generates mocks for interfaces matching this regex
- `filename: "monkey_mocks_test.go"` - Output file name
- `mockname: "monkeyMock"` - Generated mock struct name

### 4. Generate Mocks

Use the Makefile targets to generate mocks:

```bash
# Generate all mocks (including monkey mocks)
make generate-mocks

# Generate only monkey mocks
make generate-monkey-mocks
```

This will create `monkey_mocks_test.go` with the generated mock implementation.

### 5. Use in Tests

Here's how to use the monkey mocks in your unit tests:

```go
func TestYourFunction(t *testing.T) {
    t.Run("WhenSomeCondition", func(t *testing.T) {
        // Create and configure the mock
        mm := &monkeyMock{}
        mm.Test(t)
        mm.Patch()
        defer mm.Unpatch()

        // Set up mock expectations
        mm.On("InternalUtilGetSignedToken", "project-123").Return("token", nil).Once()
        mm.On("InternalUtilGetPairedRegionURI", "us-central1").Return("basePath", nil).Once()

        // Run your test
        result, err := YourFunctionUnderTest()
        
        // Assert results
        assert.NoError(t, err)
        assert.NotNil(t, result)
        
        // Verify all mock expectations were met
        mm.AssertExpectations(t)
    })
}
```

## Complete Example

Here's a complete example showing the pattern:

### `generate_mocks.go`
```go
//go:build mocks

package example

import (
    "context"
    "time"
)

type monkeyMethods interface {
    GetCurrentTime() time.Time
    ParseString(input string) (int, error)
    ValidateInput(ctx context.Context, data string) error
}
```

### `monkey_test.go`
```go
package example

import (
    "time"
    "strconv"
)

func (_m *monkeyMock) Patch() {
    GetCurrentTime = _m.GetCurrentTime
    ParseString = _m.ParseString
    ValidateInput = _m.ValidateInput
}

func (_m *monkeyMock) Unpatch() {
    GetCurrentTime = time.Now
    ParseString = strconv.Atoi
    ValidateInput = _validateInput
}
```

### Test Usage
```go
func TestExampleFunction(t *testing.T) {
    t.Run("Success case", func(t *testing.T) {
        mm := &monkeyMock{}
        mm.Test(t)
        mm.Patch()
        defer mm.Unpatch()

        expectedTime := time.Date(2023, 1, 1, 0, 0, 0, 0, time.UTC)
        mm.On("GetCurrentTime").Return(expectedTime).Once()
        mm.On("ParseString", "42").Return(42, nil).Once()
        mm.On("ValidateInput", mock.Anything, "test").Return(nil).Once()

        result, err := ExampleFunction("test")
        
        assert.NoError(t, err)
        assert.Equal(t, expectedTime, result.Timestamp)
        mm.AssertExpectations(t)
    })
}
```

## Best Practices

1. **Always use `defer mm.Unpatch()`** to ensure functions are restored even if the test panics
2. **Use `.Once()`** for expectations unless you specifically need the function called multiple times
3. **Call `mm.AssertExpectations(t)`** at the end of each test to verify all mocks were used as expected
4. **Keep the `monkeyMethods` interface focused** - only include functions you actually need to mock
5. **Use descriptive test names** that explain what scenario you're testing
6. **Group related mocks** in the same test case for better readability

## Common Patterns

### Mocking Multiple Functions
```go
mm.On("Function1", arg1).Return(result1, nil).Once()
mm.On("Function2", arg2).Return(result2, nil).Once()
mm.On("Function3", arg3).Return(result3, nil).Once()
```

### Mocking Functions with Different Arguments
```go
mm.On("GetToken", "project-1").Return("token1", nil).Once()
mm.On("GetToken", "project-2").Return("token2", nil).Once()
```

### Mocking Functions That Return Errors
```go
mm.On("ValidateInput", mock.Anything, "invalid").Return(errors.New("validation failed")).Once()
```

### Using Mock Arguments
```go
mm.On("ProcessData", mock.Anything, mock.MatchedBy(func(data string) bool {
    return len(data) > 0
})).Return(nil).Once()
```

## Troubleshooting

### Common Issues

1. **"undefined: monkeyMock"** - Make sure you've run `make generate-monkey-mocks`
2. **"function not found"** - Check that function names in `monkeyMethods` match exactly
3. **"cannot assign to function"** - Ensure function variables are declared as variables, not constants
4. **"mock expectations not met"** - Verify all expected function calls are actually made in your test

### Debug Tips

1. Use `mm.Calls` to see what functions were actually called
2. Check that `Patch()` and `Unpatch()` are called in the right order
3. Verify function signatures match exactly between interface and implementation
4. Ensure the build tag `//go:build mocks` is present in `generate_mocks.go`

## Integration with Existing Tests

The monkey mocks pattern works alongside other mocking approaches:
- Use `monkeyMock` for package-level functions
- Use regular mocks for interfaces
- Use `testify/mock` for simple cases

This approach provides a clean, maintainable way to mock functions while leveraging Mockery's powerful mock generation capabilities.
